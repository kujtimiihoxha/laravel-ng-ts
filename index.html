laravle-ng-ts

A simple starter for getting up and running with some of the most used web frameworks

Install Yeoman and the Generator

npm install -g yo
npm install -g generator-laravel-ng-ts

Prepare the project

php artisan migrate
php artisan db:seed

Create the project

mkdir my-project
cd my-project
yo laravel-ng-ts

Run and watch changes on the frontend side then run the backend server

gulp && gulp watch
php artisan serve

Now navigate to http://localhost:8000

What is laravel-ng-ts?

laravel-ng-ts is an attempt to make a good starting point for developers who want to use Angular 1.5 and Laravel 5.2. Since Angular 2 is coming in the near future this project is aiming to make the transition to Angular 2 a bit easier with typescript integration and custom decorators that use a similar syntax that Angular 2 is using..

Designed for developers

This starter is here to make the scaffolding and setup of a base skeleton for a web application easy and very fast

Time saver

Saves you time by providing generators for basic classes and files so you don't have to write boilerplate code

Pre Configured

Laravel-ng-ts is pre-configured with JWT, API, Template cashing, Sattelizer and Gulp tasks so that you can start your implementation immediately.

Live reload

This starter has live reload so you can view the changes that you write in real time and you don have to restart the server

Bower dependencies

Laravel-ng-ts is configured to automatically load bower dependencies and concatenate them for you to a single .css and .js file

Sass compiler

Sass compiler is pre configured and it will detect all the .scss files and compile them to a single .css file .

Folder Structure

Laravel

The backend part of the starter has the same folder structure as all standard laravel applications

There are just some miner changes in the resources/views folder the app/Http/Controllers folder and app/Htpp/routes.php

  • resources/views: index.blade.php
<!doctype html>
<html ng-app="app" ng-strict-di>
<head>
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1">

    <link rel="stylesheet" href="{!! elixir('css/vendor.css') !!}">
    <link rel="stylesheet" href="{!! elixir('css/app.css') !!}">
    <link href='https://fonts.googleapis.com/css?family=Lato:300,400,700' rel='stylesheet' type='text/css'>

    <title>Laravel Angular Typescript</title>

    <!--[if lte IE 10]>
    <script type="text/javascript">document.location.href = '/unsupported-browser'</script>
    <![endif]-->
</head>
<body>

<div ui-view></div>


    <script src="{!! elixir('js/vendor.js') !!}"></script>
    <script src="{!! elixir('js/templates.js') !!}"></script>
    <script src="{!! elixir('js/app.js') !!}"></script>

    {{--livereload--}}
    @if ( env('APP_ENV') === 'local' )
    <script type="text/javascript">
        document.write('<script src="'+ location.protocol + '//' + (location.host.split(':')[0] || 'localhost') +':35729/livereload.js?snipver=1" type="text/javascript"><\/script>')
    </script>
    @endif
</body>
</html>

This html file is the only file that will be returned from the server, all other views are going to be saved in the $templateCashe of the angular application

All the bower dependencies will be loaded in the css/vendor.css and js/vendor.js files

All your custom scss and the angular app (typescript) will be loaded in css/app.css and js/app.js files

The templates used by $templateCashe will be loaded in js/templates.js

AngularController

This controller is used to bootstrap the application

  • app/Htpp/Controllers: AngularController.php
    
 <?php

namespace App\Http\Controllers;

class AngularController extends Controller
{
    public function serveApp()
    {
        return view('index');
    }

    public function unsupported()
    {
        return view('unsupported_browser');
    }
}
